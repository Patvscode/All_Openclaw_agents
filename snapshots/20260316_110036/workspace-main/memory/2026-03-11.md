## What Was Built

### Knowledge Engine (port 8110) — New core service
Designed and built a 7-component knowledge platform in a single session:
1. **Layered memory**: RAG store extended with layer (raw/processed), domain, source_ref, quality_score. 17,935 existing chunks tagged raw.
2. **Enrichment pipeline**: 4 parallel workers (tagger, summarizer, classifier, linker). Each has circuit breaker + dead letter queue. 0.8b does all enrichment.
3. **Research domains**: Graph AI created as first domain. 7 concepts extracted. Domain storage in `~/.openclaw/knowledge/domains/`.
4. **Research trails**: Full audit trails with attempts, conclusions, linking. Storage in `~/.openclaw/knowledge/trails/`.
5. **Visual artifact extraction**: From PDFs (pymupdf), URLs, video frames. 4b vision describes, 0.8b tags. 3 artifacts extracted from real research PDF.
6. **Model router**: 5-tier (0.8b→2b→4b→35b→cloud). Known-route shortcuts skip LLM call. Adaptive escalation. 211ms avg routing time.
7. **Association engine**: Cross-domain conceptual linking. Batch job design. Low existing cross-domain data.
8. **Voice gateway**: STT (Whisper.cpp) + TTS (Piper fast ~1s / Sesame quality ~30s). Async API. Full pipeline: STT→route→RAG→generate→TTS.

## Key Technical Decisions
- One service (port 8110) for all knowledge features, not 7 separate services
- Researcher isolation: startup assertion in every service — ABSOLUTE
- Fire-and-forget enrichment from Memory Concierge (non-blocking)
- Piper TTS as default fast mode (~1s) vs Sesame quality mode (~30s)
- Raw data is sacred (append-only), processed data is disposable/regenerable
