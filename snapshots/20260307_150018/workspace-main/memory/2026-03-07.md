### Key Lessons Learned:
1. **85 GB model takes ~5 minutes to load on DGX Spark** — any timeout under 300s will fail
2. **systemd user services with Restart=on-failure are persistent** — disabling isn't enough if the timer also restarts them. Must disable/rename BOTH the service AND any associated timer
3. **Split GGUFs: file 1 can be tiny** — the first shard is often just metadata (11 MB). The actual weights are in subsequent shards. Point llama-server at shard 1 and it finds the rest.
4. **Port conflicts during swap are the #1 failure mode** — if the old model grabs the port before the new one starts, the new one silently fails. The supervisor must ensure the old process is fully dead before starting the new one.
5. **Unified memory means model + KV cache + OS share 128 GB** — 85.6 GB model + 32K ctx KV cache ≈ 101 GB total, leaving 20 GB. Tight but workable.
6. **Unsloth Dynamic quants are architecture-specific** — they tested 150+ configurations and 9 TB of GGUFs to find optimal per-tensor bit allocation for Qwen 3.5's hybrid Gated DeltaNet + MoE architecture
7. **The jess ecosystem has 5+ services** — q35, watchdog, keeper, memory-manager, trace-log. All need to be considered when changing the underlying model.
