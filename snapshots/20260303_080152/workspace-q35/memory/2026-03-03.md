# 2026-03-03 — Jess Memory

## 04:12 EST - P0 Memory Guardrail Fix (card-e22b85)

**Problem:** Memory guardrails (`jess_memory_manager.py`, `jess_runtime_guard.sh`) couldn't trigger because OpenClaw's session metadata returns `totalTokens: null`, making context percentage calculation impossible.

**Root Cause:** 
- llama.cpp model provider (`q35cpp`) doesn't expose total token counts
- OpenClaw session tracking doesn't accumulate tokens from all requests

**Fix Implemented:**
- Age-based triggers as PRIMARY mechanism (independent of token reporting)
- 30 min (1800s): Trigger summarize
- 60 min (3600s): Trigger summarize + cleanup + session RESET
- Context percentage retained as SECONDARY fallback (only when available)

**Files Modified:**
- `scripts/jess_memory_manager.py` - Rewrote trigger logic with age-first approach
- `scripts/jess_runtime_guard.sh` - Updated to prioritize age-based warnings

**Testing Results:**
- Runtime guard now correctly reports `level=warn` with `session_age_high:530s`
- Memory manager logs show proper action triggering based on age

**Lessons:**
- Never rely solely on `totalTokens` - it's often null across different model providers
- Age is a more reliable trigger for session hygiene than token counts
- Always verify with live checks before asserting system state
