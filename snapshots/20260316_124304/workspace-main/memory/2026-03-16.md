### Pat's Design Philosophy (CRITICAL)
The autopilot should be a **self-growing system**, not a task executor:
- Each build should make the next build smarter (compounding)
- It should build tools for ITSELF to fill capability gaps
- It should defer hard things, build prerequisites, then retry with upgraded self
- It should question its own patterns and fix them (not just journal)
- It should NOT need human interaction to improve — fully autonomous trajectory
- Humility + self-criticism + strategic patience = growth
- "It's about getting it just right so that it builds things strategically and learns"
